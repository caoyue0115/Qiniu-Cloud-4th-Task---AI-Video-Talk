# AI 视觉对话助手 — 技术实现路线

> 从零到「双引擎多模式 + 端云协同 + 无障碍」的完整技术实现
> 技术栈：FastAPI + 原生前端（HTML/JS/Web Audio/TensorFlow.js）+ 阿里云百炼通义千问全栈

---

## 一、技术选型

### 1.1 为什么这样选

| 层 | 选型 | 理由 |
|----|------|------|
| 后端 | **FastAPI + uvicorn** | 原生支持 async 与 WebSocket（实时模式中继必需）；轻量、部署简单 |
| 前端 | **原生 HTML/JS**（无框架） | 通话式单页，依赖少、加载快、易嵌入展示页 iframe；规避框架体积 |
| 语音识别 ASR | **Paraformer**（通义千问） | 中文准、支持实时与一次性；国产化 |
| 视觉理解 | **通义千问 VL（plus/max）** | 多模态强、中文好；plus 快省、max 准，配合分级路由 |
| 语音合成 TTS | **CosyVoice** + 浏览器 speechSynthesis | 云端自然 + 端侧零成本兜底 |
| 实时多模态 | **Qwen-Omni-Realtime** | 端到端音视频流→语音流，首字约 0.3s |
| 端侧检测 | **TensorFlow.js COCO-SSD** | 浏览器内实时检测人/车/红绿灯，零云端成本、毫秒级 |

### 1.2 整体架构

```
┌──────────────── 浏览器（手机/电脑） ────────────────┐
│ 摄像头 / 麦克风 / 扬声器                              │
│ VAD 静音触发 · Web Audio 16k 采样 · 震动 · 打断       │
│ TensorFlow.js COCO-SSD 端侧实时检测（导航）           │
└──────┬───────────────────────────┬──────────────────┘
   稳定模式 HTTP/SSE           实时模式 WebSocket
       ▼                            ▼
┌────────────────── FastAPI 服务端 ──────────────────┐
│ 路由层：意图分类（端侧/小/大模型/模式指令）          │
│ /api/asr  Paraformer 一次性识别（稳定可靠+重试）     │
│ /api/talk_stream  千问 VL 流式问答（SSE）            │
│ /api/scene  四大模式场景理解（不同 prompt）          │
│ /api/tts  CosyVoice 合成                            │
│ /ws/realtime  Omni-Realtime 中继（音视频流↔语音流）  │
│ 缓存 · 成本追踪 · 对话记忆（跨模式共享）             │
└──────────────────────────────────────────────────────┘
       │
       ▼  阿里云百炼 DashScope（通义千问全栈）
```

## 二、分阶段实现路线

| 阶段 | 内容 | 关键产出 |
|------|------|---------|
| **P1 核心闭环** | 语音→看图→语音 三段式打通（FastAPI + 千问 VL + Paraformer + CosyVoice）| 通话式 App、问答模式 |
| **P2 端云协同** | 意图路由、端侧颜色/检测、缓存、成本追踪、图像压缩 | 单次成本降约 62% |
| **P3 多模式** | 导航（端侧 COCO-SSD + 千问）、阅读（OCR 朗读 + 翻页）、聊天（主动话题）| 四大模式 + 语音切换 |
| **P4 无障碍** | 语音/点击打断、震动、语速、再说一遍、帮助、屏幕阅读器适配 | 全程无障碍体系 |
| **P5 实时模式** | Qwen-Omni-Realtime WebSocket 中继，端到端秒回 | 双引擎 |
| **P6 展示与交付** | 艺术化展示网站（手机样机 + 实时仪表盘）、文档、部署 | 评委可直接体验 |

## 三、关键技术实现

### 3.1 端云协同与意图路由（成本核心）

用户说话 → 路由判断走哪条路径，**简单的留端侧、复杂的才上云、能省则省**：

```
颜色/人车检测  → 端侧（TF.js / HSV）          成本 0
简单物体/文字  → 云端小模型 qwen-vl-plus       成本低
复杂场景理解   → 云端大模型 qwen-vl-max        成本高
相同问题+相近画面 → 命中缓存                    成本 0
```

### 3.2 导航模式：端侧实时 + 云端语义（混合）

- **端侧**：浏览器内 TensorFlow.js COCO-SSD，每 ~350ms 检测一帧，识别行人/车辆/红绿灯，按 bbox 位置算**方位**（左/中/右）、按面积算**远近**，**即时**语音 + 震动告警；同类同方位去重，避免唠叨。
- **云端**：千问每 ~6s 补充 COCO 检测不到的楼梯/路沿/指示牌/路况。
- **关键工程点**：端侧检测循环与麦克风**并行**，互不阻塞；模型权重**本地托管**（解决 Google 存储国内被墙）。

### 3.3 实时模式：Omni-Realtime WebSocket 中继

```
浏览器 ──(16k PCM 音频流 + JPEG 视频帧)──▶ FastAPI ──▶ Qwen-Omni-Realtime
浏览器 ◀──(24k PCM 语音流 + 文字转写)──── FastAPI ◀──（流式回调）
```

- **服务端中继**：API Key 只在服务端，浏览器不接触密钥；用线程安全队列桥接 Omni 回调线程与异步 WebSocket。
- **服务端 VAD**：开启 turn detection，模型自动判断说话结束并流式应答，自然如打电话。
- **打断**：收到 `user_speaking` 事件即停掉正在播的 AI 语音。
- **顺序约束**：Omni 要求先发音频再发图，服务端用 `audio_seen` 标志保证。

### 3.4 无障碍关键实现

- **VAD**：Web Audio 实时 RMS 检测，静音 ~0.85s 判定一句结束自动触发。
- **打断（barge-in）**：AI 朗读时持续较大音量（阈值高于静音、配合回声消除避开自我触发）判为用户打断；另有**点屏幕**兜底。
- **TTS 分级**：端侧优先（即时），无本地中文语音回退云端 CosyVoice，并有超时兜底防止状态卡死。
- **对话记忆**：稳定模式与实时模式共享同一上下文，看图问答也注入最近对话，支持"这个能吃吗"等多轮追问，跨模式不割裂。

### 3.5 展示网站（评委入口）

- `/` 展示页（静态）+ `/app` 真实 App；手机样机用 CSS + `iframe(allow camera/microphone)` 嵌入真实 App。
- 因同源，展示页直接读取 iframe 内 DOM（成本面板/对话/模式）更新左侧实时仪表盘，**App 逻辑零改动**。

## 四、部署

### 4.1 本地运行
```bash
pip install -r requirements.txt
cp .env.example .env   # 填入 DASHSCOPE_API_KEY
python server.py       # http://127.0.0.1:8000
```

### 4.2 公网/手机访问
- 临时：`cloudflared tunnel --url http://127.0.0.1:8000`（得 https 地址，手机可用）
- 正式：云服务器 + 域名 + 自动 HTTPS（Caddy/Let's Encrypt），固定地址 24h 在线。

> 摄像头/麦克风强制 HTTPS，故手机访问必须 https（本机 localhost 除外）。

## 五、目录与模块职责

| 文件 | 职责 |
|------|------|
| `server.py` | FastAPI 主服务，全部路由 + Omni 实时中继 |

| `modules/speech.py` | Paraformer 识别 + CosyVoice 合成 |
| `modules/mode_prompts.py` | 四大模式 + 实时模式的「盲人优化」提示词与意图识别 |
| `modules/color_detector.py` | 端侧 HSV 颜色识别 |
| `modules/router.py` | 意图路由 |
| `modules/cache.py` / `cost_tracker.py` / `context.py` | 缓存 / 成本追踪 / 对话记忆 |
| `utils/config.py` / `image_utils.py` | 配置与密钥 / 图像压缩 |
